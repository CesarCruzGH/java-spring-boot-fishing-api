Perfil biológico de la especie. Esta información no cambia según regulaciones.

⚠️

Consistencia de agua: Si tipo_agua = DULCE, la zona asociada en Regulación debe ser categoria_zona = CONTINENTAL.

📍 Tabla: Zona

Define dónde se aplican las reglas. Ahora combina jurisdicción, macro-región, restricción pesquera e hidrología.

🧭

Nueva estructura: la entidad Zona ahora describe ubicación operativa, tipo de restricción y contexto hídrico. Esto permite cruzar mejor especies costeras, oceánicas, estuarinas y continentales.

📍
Zona — Geografía, Restricción y Jurisdicción
8 campos · Estructura actualizada para clasificación territorial
CampoTipoDescripción y Ejemplo
id Numérico Identificador único de la zona. Ej: 1
nombre Texto Nombre oficial u operativo de la zona. Ej: Litoral Norte de Yucatán, Ría Lagartos
macro_zona Enum Región macro para ordenar reglas espaciales.
PONIENTE CENTRO ORIENTE OCEANICA CONTINENTAL

PONIENTE: desde Celestún hasta Progreso. CENTRO: de Chicxulub a Telchac. ORIENTE: de Dzilam a El Cuyo. OCEANICA: Banco de Campeche y mar abierto. CONTINENTAL: cenotes y tierra firme.
tipo_restriccion Enum Nivel operativo de restricción pesquera.
LIBRE ANP REFUGIO_PESQUERO PROHIBIDA

LIBRE: solo aplican vedas nacionales y tallas mínimas. ANP: área con plan de manejo. REFUGIO_PESQUERO: recuperación de biomasa con reglas estrictas. PROHIBIDA: veda permanente o acceso no permitido.
categoria_hidrica Enum Tipo hidrológico principal de la zona.
OCEANICA COSTERA ESTUARINA CONTINENTAL
es_anp Booleano TRUE si la zona está dentro de un Área Natural Protegida. FALSE si no lo está.
municipio_sede Texto Municipio principal asociado a la administración o acceso. Ej: Celestún, Progreso, Tizimín
notas_especificas Texto largo Observaciones especiales de manejo, acceso o restricción. Ej: Zona núcleo sin pesca, Refugio pesquero parcial temporal
⚠️

Consistencia hídrica: si una especie en la tabla Pez tiene tipo_agua = DULCE, la regulación debe apuntar a una Zona con categoria_hidrica = CONTINENTAL.

⚠️ Reglas de Oro

Normas críticas para el llenado correcto. Incumplirlas causará errores en la API de Spring Boot.

🔗
Integridad de IDs
No inventes IDs. Si registras una regulación para el "Mero", el pez_id debe coincidir exactamente con el id de ese pez en la tabla Pez.
🔢
Decimales con Punto
Usa siempre el punto (.) y no la coma (,) para medidas decimales. Correcto: 30.5 — Incorrecto: 30,5
🔠
Enums en MAYÚSCULAS
Todos los campos Enum deben escribirse en MAYÚSCULAS para que el sistema los reconozca. Ej: COMERCIAL, SALADA, FIJA.
🌊
Consistencia Agua ↔ Zona
Si el pez tiene tipo_agua: DULCE (cenotes), la zona asociada debe ser categoria_zona: CONTINENTAL. No mezclar tipos.
📋
Orden de Registro
Registra en este orden: 1. Pez → 2. Zona → 3. Regulación → 4. Veda / Arte / Similitud. Las FKs deben existir antes.
📅
Fechas como Números
En Periodo Veda, las fechas se registran como enteros. Mes: 1 a 12. Día: 1 a 31. No usar texto ni formato DD/MM.
💡

Consejo: Antes de capturar una regulación, verifica que el pez_id y el zona_id ya existan en sus respectivas tablas. El sistema Spring Boot validará las claves foráneas al momento de guardar.
